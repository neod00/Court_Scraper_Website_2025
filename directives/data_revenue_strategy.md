# SOP: 누적 데이터 기반 통계 + 수익 전략

법원 회생·파산 자산매각 공고 데이터를 매주 누적·분석하여 고유한 통계 자산을 구축하고, 단계별 수익화를 진행합니다.

---

## ⚠️ 전제조건: 데이터 축적

분기/반기/연 단위 통계를 유의미하게 만들려면 **충분한 기간의 구조화된 데이터가 필수**입니다.

- `court_notices` 테이블: 매일 스크래핑으로 원본 공고 데이터 누적 중 ✅
- `weekly_reports` 테이블: 주간 리포트 + category_breakdown JSON 누적 중 ✅
- **향후 추가 필요**: 주간 통계 스냅샷 테이블 (`weekly_stats`)
  - 카테고리별 건수, 평균/최저/최고 매각가, 지역 분포 등 구조화된 숫자 데이터
  - SQL 한 줄로 "6개월간 부동산 평균 최저매각가 추이" 같은 쿼리 가능하도록 설계
  - `court_notices` 원본이 있으므로 나중에 백필(backfill)도 가능

## 📊 만들 수 있는 통계 콘텐츠

| 통계 유형 | 예시 | 활용 방법 |
|-----------|------|-----------|
| **가격 추이** | "서울 아파트 회생매각 최저가 6개월 추이" | 블로그 자동 게시 (SEO 극대화) |
| **카테고리 트렌드** | "2026년 상반기 파산 법인 기계장비 매각 급증" | 뉴스레터/리포트 |
| **지역별 분포** | "수도권 vs 지방 매각 공고 비율 변화" | 인포그래픽 블로그 |
| **유찰률/낙찰률** | "몇 회차에서 가장 저렴하게 낙찰되는가" | **프리미엄 리포트** |
| **자산군별 할인율** | "감정가 대비 실제 최저매각가 할인율 통계" | **유료 콘텐츠** |

## 💰 수익화 로드맵

### 1단계: 무료 블로그 자동 게시 (SEO + 애드센스) ← **현재 진행 중**
- 주간 트렌드 리포트를 블로그에 자동 게시
- 매주 1편씩 SEO 콘텐츠 축적
- 구글 크롤러가 "정기적으로 업데이트되는 사이트"로 인식 → 애드센스 승인 확률↑

### 2단계: 월간/분기 심층 통계 리포트 (데이터 3개월+)
- 월간 통계 자동 생성 스크립트 개발
- 카테고리별 가격 추이 차트 자동 생성
- 블로그 + 이메일 뉴스레터 발행
- **목표**: 구독자 확보, 트래픽 증가

### 3단계: 프리미엄 구독 모델 (데이터 6개월+)
- 심층 통계 대시보드 (실시간)
- 관심 자산 알림 서비스 (Telegram/Email)
- 낙찰 사례 분석 보고서
- **수익 모델**: 월 구독 9,900원~29,900원

### 4단계: B2B 데이터 API (데이터 1년+)
- 법무법인/금융사 대상 데이터 API 판매
- 커스텀 리포트 생성 서비스
- **수익 모델**: 월 API 사용료 50만원~

## 📦 데이터 인프라

### Supabase 저장 구조

| 테이블 | 용도 | 1년 예상 용량 |
|--------|------|---------------|
| `court_notices` | 개별 공고 데이터 | ~50MB |
| `weekly_reports` | 주간 리포트 (52건/년) | ~260KB |
| `monthly_stats` (향후) | 월간 통계 집계 | ~60KB |
| `price_history` (향후) | 가격 추이 시계열 | ~5MB |

**총 예상 용량**: 연 ~60MB 이내 (Supabase 무료 플랜 500MB 내 충분)

## 🔧 구현 우선순위

1. ✅ 주간 리포트 Supabase 누적 저장 (완료)
2. 🔄 주간 리포트 → 블로그 자동 게시 (1단계 진행 중)
3. ⬜ GitHub Actions 주간 cron 등록
4. ⬜ 월간 통계 집계 스크립트 개발
5. ⬜ 가격 추이 시계열 데이터 수집
6. ⬜ 프리미엄 대시보드 설계

## 📌 참고사항

- `weekly_reports` 테이블은 `UNIQUE(week_start, week_end)` 제약으로 같은 주 중복 방지
- 다른 주의 데이터는 새 행으로 추가되어 자동 누적
- 1년 52건 × 약 5KB = 약 260KB로 용량 부담 없음
